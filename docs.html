LooWID Project

LooWID is an open source project that provides a way to perform video conversations (audio and video streaming) and sharing their desktop windows with other people using web browsers. You can build your own instance or access to this service through www.loowid.com.

In order to use the platform you must create a room and then send or share the unique URL provided by the service to let other people join the conversation. The method to send the URL must be done out of the www.loowid.com. It could be done sharing by email, twitter, facebook, etc. Each person that opens the URL in the browser will enter to the room and see and listen everything that would be streamed in the room.

Nobody has to register or install nothing in your computer, just access to the URL with the browser.

Getting started

Once you acceded to www.loowid.com or your own LooWID instance, you'll find the landing page screen. You can choose your hero name, write yours or select another aleatory name clicking on the hero's face. Also you'll find a link to the terms of service. They must be accepted to create the room. Once you got the desired name Click on "LooWID now" button.

After some seconds your LooWID room will be available. Congratulations you are a new room owner. If it’s the first time you enter to the room, you will be asked to grant Notification permission. It’s recommendable you accept it, but we’ll talk about them later.

In the room you'll see the following elements:

Main Media Area

Here you’ll have all participant’s media windows. You’ll be able to move and resize each window for all this area.

Control Bar

In this bar you’ll find the buttons to activate desktop, video, and audio capture. There are also buttons to access to the room configuration (as room owner) and the audio alerts activation (just in case you blocked Notifications on desktop). At the right of this bar you’ll see the button to leave the room.

Connections status panel

At the left of the room, you’ll have the list of participants in the conversation. This list will be useful to perform actions with users like sharing files, change profile information, ask for sharing a participant device, and eject whoever you want.

Chat panel

There is a chat to write messages to the other participants. Particularly useful when owner handles the session and the others maintain their microphones off.

Both panels, connections and chat, can be collapsed to get a bigger video area, pressing the arrows that are at the top of each one.

Start your meeting

Share the URL

The first thing to do is share/send the URL of your room. You’ll find at the center of the Main Media Area, or on the Control Bar. Click with right-button and copy the URL and spread it with your guests.

As soon they connect to connect to the room they will be appearing in the Connections panel. Note that if they’ve never run LooWID they will be show with an aleatory superhero name, and default image. You and your guest can change this information clicking in the image, but we’ll see in detail later.

Stream your media

It’s time to stream your media (webcam, audio or desktop). Click on the buttons of the head control bar to star. Note that first time browser will ask for permission to capture audio or video from your devices. You must accept to go on.

You can use the same buttons to stop sharing or just close the window. Audio is not activated by default with the cam, so you must remember to activate it also. We will see more details about sharing desktop below.

Viewer participants

When some of your guest shares his video, a window is opened automatically in your Media Area. You can ask them for sharing but let’s see it below when we talk about room options. As owner you’ll be able to eject from the room and send files to them. There are two buttons for each guest to do that.

Send files

To send a file click on the cloud icon of participant who you want to send and select the files. If participant accepts those files you’ll start sending them directly from browser to browser. Once you started an upload, you can cancel it. When file transfer raises to 100% participant will be able to open or download the file to his hard-drive.

Now that you know the main elements, let’s talk of some options you can configure of your room.

Change your information

Participants can change their names and pictures. To change your information click on your pictures at Connection status panel and enter your name and your email. Picture is get from Gravatar profile, be sure that you write the email address registered in your Gravatar account.

Room configuration

As I explained above, you’ll see the options button in the top header bar. The main options you can configure are:

Title

You can set a title for the room, it will be shown at control bar at the left of the room's URL.

Standing room

By default when you create a room is volatile. It means that when you (owner) exits, the room is destroyed and can’t be acceded again. Maybe you need to have a room for some time. Mark this options and the room will be active for some time (15 days). That would help you to plan a meeting and share the URL with participants some time before.

When you configure the room as standing room a card with an URL address will be show. This address is the one that let you come back to the room as owner.

Take care about the room won’t be active unless the owner is not present. Remember to save that URL in a safe place because if you lose it you’ll not be able to access and the room will be lost.

Moderate room

This options allow you to handle room as moderator. By default all participants of the room can share their media without permission, but there are some scenarios where you would prefer to select who and when can share their media. If you select this options users won’t have the sharing buttons in the head bar, instead they could ask for sharing pressing the icons below their user in connections status bar. If you accept the request they’ll share the media. At same time you can initiate and ask to specific viewer to share a media (video, audio or desktop).

Chat disabled

If you want to be the only person who can write to the chat, mark this option. Useful if you want to write some instructions before the session starts. You can enable again in any moment of the session.

Lock room

Maybe you are not interested in allow to access more people to your room. When this option is selected users that aren't yet in the room won’t be able to access.

Access permission.

By default someone with the link of the room would be able to access it when owner is in it. If you want to add an extra security access step can protect your room with a password. Once guests access to the room URL password will be asked. Just those who know it will enter.

Sharing your desktop

Share desktop, just now, is a feature that is only available if you are using Chrome on a Mac, Windows or Linux. In order to do that you will also need to install a Chrome extension. The easiest way to do that is:

1- Click on “Share desktop” button. A message will appear at the bottom of the screen telling about installing this extension. At the end of the message you’ll see the link to the Chrome store to the loowid extension.

2- Follow the link, and install the extension.

3- Press the reload browser’s button or just press F5.

4- Press again the share desktop button. After some seconds the extension will ask you what to share. You can choose between full desktop or a single window.

Selecting the window of your slides would be a valid option to do an online presentation.

Using the chat

Your room’s chat can be used for many purposes. We’ve already seen how to disable the chat from options, but it will be an useful tool to let guest participate in the session, specially when you are not seeing the rooms page.

Notifications

As I explained before when you access to the room browser requests you permission to access to Notification. That means that browser can show you Notifications even if you don’t have the focus on the room page. When you accept messages in your desktop will be displayed when people enter and exits of the room or writes in the chat.

If you don’t want that kind of notifications (maybe you want to share your full desktop and don’t want to show that notifications) you can’t either don’t accept the notification permission when browser asks for them or disabling on browser any time.

Audio notifications

If you don’t allow Notifications audio notifications will be enabled.

Once you disabled Notifications chat messages are spoken when room is not focused or chat is collapsed. It’s an alternative mechanism to be alerted from your guest (questions, suggestions, …) when you are not looking to the chat. This is a common situation if you are doing are explaining something sharing an application window. You can either disable that feature pressing the bell icon at the head controls bar.

Play videos in the chat

You can also use chat to share YouTube videos and play them on a window. Just type a video's URL in the chat and send it. Everybody will see the video preview in the chat panel. When a participant click on the preview the YouTube video will be opened in a window.

LooWID's communication stack

As said before on this wiki LooWID is a web video conferencing platform based on WebRTC that uses peer-to-peer connections to send video and audio directly from browser to browser. Although WebRTC let browsers exchange data between peers, some servers are needed. As we saw before signaling and room management is done in LooWID, but there are other services needed to help WebRTC to work throughout NAT. At this point ICE, STUN and TURN start to play. ICE framework uses STUN and TURN to establish a connection (RTCPeerConnection) through any NAT variety. ICE tries to connect to peers directly via UDP. STUN server tries to find the public address and port. If UDP fails, ICE tries TCP.

If finally communication is not possible (firewalls, restrictive NAT… ) then ICE relays on a TURN server to perform the data exchange.

By default, LooWID uses generic STUN servers from Google, but it would be recommended that if you are planing to run your own server use a set of you own.

In order to simplify configuration we added support to configure a Xirsys account. They provide an authenticated STUN/TURN service on the cloud. Once you set up your account and service you only need to configure the following environment variables:

XIRSYS_DOMAIN=<xyrsis domain> XIRSYS_USER=<xyrsys usename> XIRSYS_SECRET=<xyrsys password>

How to Install

The first you need to do is install nodeJS, git and mongodb. Take a look to these tutorials to know how to do it in your SO: http://git-scm.com/book/en/v2/Getting-Started-Installing-Git Install MongoDB * http://nodejs.org/

Now download LooWID source code with git client from a system console

  `git clone https://github.com/loowid/loowid /install/dir`

Once you've got the code, we recommend to install a cert. If you want it for a valid domain you can get one from http://www.cert-depot.com/ , but you can also run it with self-signed one. Save them as private.pem and public.pem in the root app directory.

If there isn't certificate available loowid startup in http port. When https port is running, http port is used only to redirect to https port.

Now, you can install all the stuff necessary to run the LooWID, just execute the following commands:

  • npm install --production
  • npm start

Ready to run:

Connect to https://localhost/ or https://your-domain/ and enjoy!

Note: If you want to work with the latest stable release then you must clone https://github.com/loowid/loowid/tree/1.1.0

Configuration

There are some environment variables you may set to configure your loowid deployment. Check your hosting provider documentation in order to know how to set this values, for example, with openshift you can type: rhc env set VARIABLE=VALUE -a app if you run in linux you can set them using:

export VARIABLE=VALUE

  1. Credentials to access to some admin resources (debug level and stats). By default admin/admin.

    ADMIN_USERNAME=<your-admin-username> ADMIN_PASSWORD=<your-admin-password>

  2. Chrome plugin extension id. Set the id of your own plugin extension for screen sharing. A generic domain plugin will be used by default.

    CEXTID=<generic-domain-plugin-id>

  3. Enable [STUN/TURN service](https://github.com/loowid/loowid/wiki/LooWID's-communication-stack) on XirSys.

    XIRSYS_DOMAIN=<xyrsis domain> XIRSYS_USER=<xyrsys usename> XIRSYS_SECRET=<xyrsys password>

  4. Room timeout. The number of days that a room can be active on the server before expires (15 days by default).

    ROOM_TIMEOUT=15

  5. Path to certificate files. By default public.pem and private.pem in the same folder, you could set the full path to your own certificate files.

    PUBLIC_KEY=<path-to-your-cert-public.pem> PRIVATE_KEY=<path-to-your-privatekey.pem>

  6. LTI Producer Configuration. The consumer connects with multiple LooWID rooms depends on LTI context, the first user with OWNERROLE will be the room owner. The loowid producer entrypoint is configured with LTIPATH (/lti). Set LTI_DOMAIN if your host is behind a proxy with other domain than req.headers.host. See more about LTI

    LTI_KEY=<lti-key> LTI_SECRET=<lti-secret> LTI_PATH=<lti-url> LTI_DOMAIN=<lti-host> LTI_OWNER_ROLES=<lti-owner-role1>,<lti-owner-role2>,...

  7. Control host and port for websocket connection.

    WS_HOST=<websocket-host-server> WS_PORT=<websocket-port-server>

  8. MongoDB URL. By Default loowid uses localhost to connect with mongodb, set one of this variables to connect to another URL.

    MONGOLAB_URI=<MongoDB URI> MONGOHQ_URL=<MongoDB URL>

Setting up IMS LTI connection

Maybe you want to use your LooWID server for education and you'd like to use it with the LMS (Learning Management System) of your institution. You are lucky because LooWID has the ability to communicate with those systems like Moodle, Sakai, Canvas, Blackboard, Drupal... using the standard IMS LTI (IMS Learning Tool Interoperability) v1.1

What is IMS LTI?

IMS LTI is an standard that allows to integrate remote learning applications with LMS, portals or educational environments. Those applications are called LTI producers. Systems that uses that applications are called LTI consumers.

If you want to go deeper in the standard I recommend to see this Dr.Chuck Severance's video explaining

How to enable LTI on LooWID

The consumer connects with multiple LooWID rooms depending on LTI context, the first user with OWNER_ROLE will be the room owner. You must set the value of the environment variables.

LTI_KEY=<lti-key> LTI_SECRET=<lti-secret> LTI_PATH=<lti-url> LTI_DOMAIN=<lti-host> LTI_OWNER_ROLES=<lti-owner-role1>,<lti-owner-role2>,...

  • LTI_KEY: A key for identify your service
  • LTI_SECRET: The secret to authenticate the service connection
  • LTI_PATH (optional): Relative endpoint path where the LTI service is listening.
  • LTI_DOMAIN: An alternative host name where your service is responding. Useful if your LooWID instance is running in under special proxy configuration.
  • LTIOWNERROLES: A ',' separated list with the role names of the LMS that can be as LooWID room owner.

How do I configure my LMS.

Depending on the LMS where you want to add LTI connection you will have follow different steps in order to fill LTI key, LTI secret and the launch URL.

The first two parameters respond to the ones you configured in the previous step. The launch URL will be https://<your-loowid-intance-domain>/lti . Take care if you changed the parameter LTI_PATH it should be the path of the launch url. You should check the option that allows sending names and emails to the tool. It's necessary to get real names and avatars.

You can see more detailed information on:

  • How to configure an external tool in Sakai
  • How to configure an external tool on Moodle
  • How to configure an external tool on Canvas
